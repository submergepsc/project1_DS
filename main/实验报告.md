## 实验目的
- 通过 Qt Widgets 图形界面与自定义链式存储结构的结合，训练综合运用数据结构与面向对象编程的能力。
- 熟悉 Qt 信号与槽机制，掌握界面事件驱动程序的组织方式。
- 针对多项式解析、运算与求值等典型算子开展算法设计与健壮性分析。

## 需求分析
1. 支持用户以字符串形式输入多项式 A、B，系统应能容错处理空白、缺省系数、括号包裹指数与显式正负号等多种输入模式。
2. 界面需完成多项式的构建、显示、加法、减法，以及 A(x)、B(x)、(A+B)(x)、(A-B)(x) 的求值操作，所有结果以文本形式输出。
3. 当出现非法输入、缺失必要括号、指数为负或自变量为零导致除零等情况时，应及时给出提示信息并阻断错误结果。

## 系统设计概述
- **界面层**（`mainwindow.ui` + `MainWindow`）：负责收集输入、展示结果，借助信号与槽把用户操作映射为多项式对象上的方法调用。
- **业务逻辑层**（`Polynomial` 类）：封装链式稀疏多项式的增删查、运算、求值以及字符串解析逻辑，保证对外呈现的都是排好序且合并后的多项式。
- **数据存储层**（`Polynomial::Term`）：以降幂单链表作为底层表示，每个结点保存系数、指数和 next 指针，保持同指数项合并后的稀疏结构。

## 关键算法与代码分析
### 1. 字符串解析流程
`Polynomial::parseFromString` 对输入多项式进行词法分析并构建链表，其核心步骤如下：
1. **预处理**：去除所有空白字符，并调用局部 `stripParens` 函数剥离首尾成对的 ASCII 或全角括号，便于处理用户以 `( ... )` 录入的复合表达式。【F:polynomial.cpp†L36-L63】
2. **项级解析**：循环扫描串中每一项，支持项首 `+/-` 号；若缺省系数则默认为 1 或 -1；检测变量符号 `x/X` 并默认指数为 1。【F:polynomial.cpp†L69-L124】
3. **指数解析**：遇到 `^` 进入指数状态机，允许指数被括号包裹并携带显式符号，严格校验括号闭合及指数至少包含一位有效数字；对非法情况即时 `clear()` 并返回失败，从而阻止残余无效结点残留。【F:polynomial.cpp†L101-L147】
4. **结点插入**：调用 `insertTerm` 将非零项写入链表，保证按降幂排序并自动合并同类项；若项后字符不是 `+/-` 则判断为语法错误。【F:polynomial.cpp†L149-L172】【F:polynomial.cpp†L260-L332】

整体上，解析器在读取指数前后都伴随多处边界条件判断，保证“读一个失败一个”而不会污染已有数据。若整串为空则视为零多项式以方便初始化界面。

### 2. 降幂链表存储与插入策略
链表节点 `Term` 包含 `coefficient`、`exponent` 与 `next` 指针，所有节点按指数降序排列。`insertTerm` 函数承担以下职责：
- 若链表为空或首节点指数更小，则直接头插；否则在遍历中保持 `previous` 与 `current` 双指针寻找插入位置。
- 当检测到相同指数时，将系数叠加，并在叠加结果为零的情况下安全删除旧节点，保证稀疏性与有序性。【F:polynomial.cpp†L274-L332】

这一策略使解析器、加减运算都能复用插入逻辑，避免在外层重复排序及合并。

### 3. 多项式加减运算
`add`、`subtract` 函数基于双指针归并思想逐项合并两个有序链表：
- 指数较大的先写入结果；指数相等时执行加减并跳过系数相抵消的项；
- 单个链表遍历完成后，将另一个链表剩余结点依序插入。

该算法时间复杂度为 O(m+n)，其中 m、n 分别为两个多项式项数，且直接复用 `insertTerm` 维持输出有序。【F:polynomial.cpp†L204-L258】

### 4. 数值求值与快速幂
`evaluate` 遍历链表并调用静态 `power` 函数计算 `x^k`：
- `power` 使用二进制快速幂迭代求值，并通过 `remaining` 的位操作实现 O(log|k|) 的复杂度；
- 对负指数分支，先将指数转为正数累乘，再在返回前取倒数；若结果为 0 则返回正无穷，后续界面可通过判断提示用户除零风险。【F:polynomial.cpp†L234-L258】【F:polynomial.cpp†L332-L365】

### 5. 界面交互与状态管理
`MainWindow` 构造函数中集中完成所有按钮与槽函数的绑定，使界面行为在代码层一目了然。【F:mainwindow.cpp†L13-L24】槽函数的主要职责如下：
- `buildPolynomialA/B`：调用解析器构建多项式并在输出区域显示表达式与稀疏序列，借助 `hasA/hasB` 标志位保证后续操作的合法性。【F:mainwindow.cpp†L43-L83】
- `addPolynomials`、`subtractPolynomials`：在检测 A、B 均已建立后执行运算并展示结果，体现出“业务逻辑由 `Polynomial` 提供，界面只负责调用”的分层思想。【F:mainwindow.cpp†L97-L124】
- `evaluatePolynomial*`：通过 `readValue` 统一解析 `x` 的输入，并利用 `formatNumber` 控制输出精度；计算组合多项式时临时构造新对象再求值，避免污染原始多项式状态。【F:mainwindow.cpp†L126-L178】

此外，所有错误提示均以中文输出，使实验报告的用户体验分析部分有现实依据。

### 6. 异常与边界情况处理
- 输入为空：解析函数视为零多项式，界面仍提示构建成功，从而允许用户逐步完善输入。
- 仅符号无系数：自动补全为 ±1，符合数学惯例。
- 指数括号缺失或只含符号：立即失败并清空链表，避免残留脏数据影响后续运算。
- 负指数且 `x=0`：快速幂返回无穷大，可在界面层追加“结果无穷大或输入非法”的提示。
- 多项式显示：`toExpressionString` 自动省略系数 1/−1 与指数 1，确保输出简洁；`toSequenceString` 提供项数与系数指数对，便于调试与验证。【F:polynomial.cpp†L174-L233】

## 实验步骤
1. 使用 Qt Designer 搭建主界面，配置多项式输入框、`x` 值输入框、运算按钮与输出文本框。
2. 实现 `Polynomial` 类的链表结构与基础操作，先通过单元测试（命令行）验证插入、加减、求值的正确性。
3. 编写 `parseFromString` 解析器，分阶段调试：先支持整数正指数，再逐步加入括号、显式符号与负指数处理。
4. 将多项式逻辑整合进 `MainWindow` 槽函数，串联起界面输入、解析、运算与输出流程。
5. 在实际运行中验证各种边界输入（空串、含空格、指数括号、负指数、指数与系数抵消等），确保提示信息准确。

## 功能演示与测试
- **样例 1（基础加减）**：
  - A：`3x^4-5x^2+7`
  - B：`x^3-2x`
  - 期望输出：`(A + B)(2) = 42`，`(A - B)(2) = 34`
- **样例 2（括号与负指数）**：
  - A：`(2x^(-2)) - x + 1`
  - B：`-3x^(+3) + x`
  - 输入 `x = 1` 时，程序应提示 `(A - B)(1)` 结果为 `6`，并正确识别括号与正号。
- **样例 3（非法指数）**：
  - 输入 `x^(+)` 或未闭合括号 `x^(2`，界面应显示“多项式输入无效”，`hasA/hasB` 置为 false。

## 附录与使用说明
1. **编译方法**：使用 Qt Creator 打开项目根目录的 `CMakeLists.txt`，配置好 Qt 5.15.2 Kit 后构建；或在终端执行 `cmake -B build -S .` 与 `cmake --build build`。
2. **运行方法**：在 Qt Creator 中直接运行，或进入 `build` 目录执行生成的可执行文件。
3. **界面操作流程**：
   - 在多项式输入框中录入形如 `3x^2-5x+(2)` 的字符串，点击“构建 A/B”。
   - 点击“显示”按钮可查看表达式与稀疏序列两种表示。
   - 在数值输入框填写 `x`，选择 A、B 或组合多项式按钮即可获得对应结果。
4. **后续扩展建议**：
